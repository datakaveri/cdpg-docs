Intelligent Universal Data Exchange

An open-source platform for the secure, authenticated, and consent-driven exchange of data between providers and consumers — an open standard (BIS IS 18003) deployed across India's smart cities and beyond.

Why IUDX

Open, standards-based, sovereign data exchange

IUDX interconnects data silos through open APIs and consent, so providers retain full control over their data.

»

Open source — a freely available Digital Public Good with full ownership and no lock-in.

»

A national standard — adopted by the Bureau of Indian Standards as IS 18003.

»

Consent-driven and auditable — data is shared only with explicit provider consent, and every access is logged.

»

Federated by design — independent and third-party resource servers coexist in one exchange.

Key features

What a secure, open data exchange needs

Standards-based APIs, discovery, consent, and governance — delivered as modular, cloud-native microservices.

APIs

Open, standards-based APIs

Resource-access APIs harmonized with ETSI NGSI-LD, plus support for OGC, GTFS, FHIR, and RESTful APIs for interoperability across legacy and modern systems.

Discovery

Catalogue & discovery

A Catalogue Server indexes datasets, APIs, and metadata as JSON-LD, with geo-spatial, attribute, and text search so consumers can find relevant resources.

Trust

Consent & access control

Dynamic, user-defined access policies, consent-driven sharing, a runtime policy engine and authorization services, and audit logs for full traceability.

Ingestion

Data ingestion

Pluggable adaptors for APIs, files, IoT, and message queues, with secure onboarding and validation pipelines for diverse data sources.

Architecture

Federated & cloud-native

A modular microservices architecture with independent and third-party resource servers coexisting in one exchange — for centralised or federated deployments.

Security

Security & observability

TLS everywhere, certificate-based authentication, API gateways for traffic control and rate limiting, immutable audit logs, and full-stack monitoring.

Data Access Models

Three ways to access data, one trust fabric

IUDX supports bulk downloads, pull-based APIs, and push-based streams — all governed by consent and policy.

Bulk Download
A copy of a dataset is downloaded to authorized users — ideal for static, historical, and large-scale offline analysis.
Pull-based API
Regulated, policy-based, on-demand access on a per-record basis through secure, standardized NGSI-LD APIs.
Push-based Stream
Real-time updates streamed to subscribed consumers via the data broker — for IoT, time-series, and live events.
