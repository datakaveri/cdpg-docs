# Definitions and Acronyms

| Term | Definition |
| --- | --- |
| **Data Provider** | A legal entity responsible for providing data for resources, providing metadata for resources, and providing authorization to use resources. |
| **Data Consumer** | A legal entity that consumes a resource via a web or mobile app. Responsible for obtaining consent and/or access for the resource and binding to the Data Provider's legal agreement for usage. |
| **Delegate** | A legal entity that can act on behalf of a Data Provider or Data Consumer, binding to the defined legal agreement and adhering to enforced restrictions. |
| **Data Principal** | A legal entity that has associated Personally Identifiable Information (PII) with a data resource. |
| **Personally Identifiable Information (PII)** | Any information that can be used to identify a Data Principal directly or indirectly. |
| **IUDX system** | A service that hosts a set of interfaces allowing Data Providers, Data Consumers, and Delegates to interact. |
| **Catalogue Service** | A registry of metadata about the resources available in IUDX for consumption and usage. |
| **Authorization and Identity Service** | A policy and token-management service available in IUDX. |
| **API Gateway** | A software component that connects with Resource Servers belonging to Data Providers for data access, per the requests of Data Consumers. |
| **Resource Server** | An external service owned and managed by Data Providers. It holds the data of a resource and exposes an interface for the API Gateway. |
| **Authorization Token** | A machine-readable token that allows Data Providers and Data Consumers to prove their identity and access. Generated by the Authorization and Identity Service after verifying consent rules, and presented to the relevant DX APIs for access. |
| **Resource** | An entity in a data-exchange system uniquely identified by an `id`. When queried, it responds with data per the data descriptor defined in the Data Explorer. |
| **Dataset** | An entity containing one or more resources belonging to the same Data Provider. |
| **Data** | The response provided by a resource when queried, represented in standardized formats such as JSON or XML. |
| **Metadata** | Information about data generated by a resource, described in a standardized, interoperable format explaining the meaning and definitions of responded attributes. |
